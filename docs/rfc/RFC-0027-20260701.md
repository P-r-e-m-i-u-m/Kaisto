## Summary

This RFC proposes implementing distributed tracing using OpenTelemetry to gain end-to-end visibility into request flows across all service components.

## Motivation

Root cause analysis during incidents requires manual log correlation. Cannot measure latency contribution of individual components. Debugging distributed failures takes 3-4x longer than necessary.

## Detailed Design

Auto-instrument HTTP, PostgreSQL, Redis and fetch. Export pipeline: OpenTelemetry Collector to Jaeger (dev) and Datadog APM (prod). Custom spans for business logic critical paths.

## Alternatives Considered

Custom correlation IDs only: already implemented but insufficient. Datadog APM proprietary SDK: vendor lock-in. AWS X-Ray: AWS-specific not portable.
