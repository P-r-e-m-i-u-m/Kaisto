# ADR-0028: Use PostgreSQL JSONB for Flexible Metadata Storage

**Status:** Accepted
**Date:** 2026-07-09
**Author:** P-r-e-m-i-u-m
**Reviewers:** Engineering Team

---

## Context

Different user types in our system require different sets of attributes. Using fixed schema columns required frequent migrations and caused sparse rows.

## Decision

We will use PostgreSQL JSONB columns for user metadata with GIN indexes for efficient querying. Fixed schema columns will be retained for fields used in WHERE clauses.

## Alternatives Considered

| Option | Pros | Cons | Decision |
|-|-|-|-|
| Chosen approach | Best fit for requirements | Some tradeoffs | Accepted |
| Alternative A | Simpler | Missing key features | Rejected |
| Alternative B | More features | Too complex / costly | Rejected |

## Consequences

### Positive
Zero schema migrations needed for new user type attributes. GIN indexes provide sub-10ms JSONB queries even at scale.

### Negative
JSONB columns cannot be used in foreign key constraints.
